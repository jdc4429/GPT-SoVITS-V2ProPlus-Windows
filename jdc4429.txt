Added requirements.txt files for easier install.  Qwen3-TTS can now run under GPT-SoVITS environment. Made changes to fix them working together.

Install in Powershell Admin CMD promt.
Use powershell script from inside conda env - 'SoVITS': install-FINAL-FIXES.ps1

The script may fail with an error stating requirements.txt failed but in my case it did install correctly, I just reran the script again to get to completion message. Doing pip install requirements did not make any changes or give any errors

Installed with CU126

conda create -n SoVITS python=3.10
conda activate SoVITS
powershell -File install-FINAL-FIXES.ps1 -Device CU126 -Source HF   (Can select from CU124, CU126, CU128)
conda deactivate


To fix errors:
I am also getting this error:
ffmpeg.exe - Entry Point Not Found - The procedure entry point libintl_bind_textdomain_codeset could not be located in the dynamic link library C:\Users\jeffc\anaconda3\envs\SoVITS\Library\bin\gdk_pixbuf-2.0-0.dll.

This is a DLL version conflict — ffmpeg.exe is trying to load gdk_pixbuf-2.0-0.dll from your conda environment, but that DLL was compiled against a newer version of libglib that exports libintl_bind_textdomain_codeset differently. Here's how to fix it:

Fix 1: Downgrade libglib in your SoVITS environment (Most Reliable)
The root cause is that libglib=2.78.4=h55e6270_1 on Windows causes this symbol mismatch. The fix is to pin it to the *_0 build.

conda activate SoVITS
DIDN'T WORK - conda install libglib=2.78.4=*_0 -c conda-forge
WORKING FIX - conda install libglib=2.78.4=*_0 gdk-pixbuf -c conda-forge --force-reinstall
